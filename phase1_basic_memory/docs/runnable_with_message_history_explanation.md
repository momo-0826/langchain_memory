### `RunnableWithMessageHistory` 詳細解説：デュエルプロファイラーの仕事

`RunnableWithMessageHistory`は、一見すると複雑な魔法のように見えるかもしれない。だが、その正体は、**お前の代わりに働く、非常に几帳面で仕事ができるアシスタント**のようなものだ。

このアシスタントの名前を、仮に「**デュエルプロファイラー**」と呼ぼう。

お前が作った`runnable`（`prompt | llm | parser`の部分）が、実際に戦う**デュエリスト**本体だ。デュエリストは、渡された手札（プロンプト）でどう戦うかしか考えていない。過去のターンのことなど覚えていない。

**デュエルプロファイラー（`RunnableWithMessageHistory`）**は、そのデュエリストの隣に立ち、試合の全てを記録し、次の手を打つ前に過去の記録をデュエリストにそっと渡してくれる、最高のサポーターなんだ。

では、お前が`with_message_history.invoke()`を呼び出した瞬間から、このプロファイラーが内部で何をやっているのか、その一挙手一投足を分解してみよう。

---

### `invoke`が呼ばれた瞬間の内部挙動（プロファイラーの仕事）

お前がこのコードを実行する：
`with_message_history.invoke({"input": "俺の名前がわかるか??"}, config={"configurable": {"session_id": "session_1"}})`

この時、**デュエリスト(`runnable`)**にはまだ何も届いていない。まず、**デュエルプロファイラー(`RunnableWithMessageHistory`)**が、その呼び出しを全て受け止める。

**ステップ１：【プロファイラーの介入】セッションIDの確認**

*   プロファイラーは、まず`config`の中身を見て、`session_id`が`"session_1"`であることを確認する。
*   「なるほど、今回は『session_1』のデュエルログを扱えばいいんだな」と理解する。

**ステップ２：【プロファイラーの仕事①】過去のログ（履歴）の取得**

*   プロファイラーは、受け取った`session_id`を使って、お前が定義した`get_session_history("session_1")`関数を**自動的に呼び出す**。
*   この関数は、`store`辞書から`"session_1"`に対応する`ChatMessageHistory`オブジェクト（過去の会話が詰まったリスト）を取り出す。
*   仮に、その中身が `[HumanMessage(content='俺の名はジャックだ。'), AIMessage(content='ジャック、よろしく。')]` だったとしよう。プロファイラーは、このリストをそっくりそのまま手に入れる。

**ステップ３：【プロファイラーの仕事②】デュエリストへの「手札」の準備**

*   プロファイラーは、デュエリスト(`runnable`)に渡すための、完璧な「手札」（入力データ）を準備する。
*   まず、お前が`invoke`で渡したオリジナルの入力データ `{ "input": "俺の名前がわかるか??" }` を用意する。
*   次に、ステップ２で取得した過去の履歴リストを、`history_messages_key`で指定されたキー名（`"history"`）を使って、その辞書に**合体（assign）**させる。
*   結果として、プロファイラーの手元には、次のような**新しい辞書**が完成する。
    ```python
    {
        "input": "俺の名前がわかるか??",
        "history": [
            HumanMessage(content='俺の名はジャックだ。'),
            AIMessage(content='ジャック、よろしく。')
        ]
    }
    ```

**ステップ４：【デュエリストの出番】`runnable`の実行**

*   プロファイラーは、この完璧に準備された新しい辞書を、ようやくデュエリスト、つまりお前が作った`runnable`（`RunnablePassthrough.assign(...) | prompt | llm | StrOutputParser()`）に渡す。
*   `runnable`は、この辞書を受け取って、初めて仕事を始める。
    *   `RunnablePassthrough`が`input`をパスし、
    *   `prompt`が`history`と`input`の両方を使って、LLMへの完全な指示書を作り、
    *   `llm`が応答を生成し、
    *   `StrOutputParser`がそれを綺麗な文字列にして、結果をプロファイラーに返す。
*   デュエリストの仕事はここまでだ。

**ステップ５：【プロファイラーの仕事③】今回のターンの記録**

*   プロファイラーは、デュエリストから返ってきた応答（例：`"もちろん、君の名前はジャックだ。"`）を受け取る。
*   プロファイラーは、今回のやり取りを忘れないように、デュエルログに追記する。
    *   お前の入力：`HumanMessage(content='俺の名前がわかるか??')`
    *   AIの応答：`AIMessage(content='もちろん、君の名前はジャックだ。')`
*   この２つの新しいメッセージを、再び`get_session_history("session_1")`を使って`store`の中にある履歴リストの**末尾に追加**する。

**ステップ６：【最終報告】結果を返す**

*   プロファイラーは、全ての裏方の仕事を終え、デュエリストから受け取った最終的な応答文字列だけを、何事もなかったかのように、お前（`invoke`の呼び出し元）に返す。

### イメージの再確認：墓地活用と手札増強

*   **デュエリスト（`runnable`）**であるお前は、毎ターン**`"input"`という新たなカードをドローする**。
*   **デュエルプロファイラー（`RunnableWithMessageHistory`）**は、お前の**墓地（`store`）**から**過去のやり取り（`history`）を全て拾い上げ**、お前の手札に強制的に加えさせる。
*   お前は、そのターンの**新しいカード（`input`）**と、墓地から拾った**過去のカード全て（`history`）**を組み合わせた完璧な手札で、そのターンの最適な一手を打つ。

`RunnableWithMessageHistory`は、まさに**墓地活用と手札増強を自動で行ってくれる、最強のサポートカード**というわけだ。
